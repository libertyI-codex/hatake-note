# 畑ノート ローカル版 Ver.1.0

家庭菜園・畑・花壇を、スマホで区画ごとに管理するためのローカルWebアプリです。

このリポジトリは **Ver.1.0 試作5** です。試作1の区画管理、試作2の作業記録、試作3の育成スケジュール、試作4の写真記録に加えて、編集・削除機能を追加しています。

## 起動方法

`hatake-note-local/index.html` をブラウザで開いてください。

PCで確認する場合は、エクスプローラーから `index.html` をダブルクリックして開けます。

スマホで確認する場合は、同じファイル一式をスマホで開ける場所に置くか、PC上で簡易サーバーを起動して同じWi-Fiからアクセスします。

## 保存方法

区画データはブラウザの `localStorage` に保存します。

保存キーは以下です。

```text
hatakeNoteLocal.plots.v1
```

作業記録データはブラウザの `localStorage` に保存します。

保存キーは以下です。

```text
hatakeNoteLocal.workRecords.v1
```

育成スケジュールデータはブラウザの `localStorage` に保存します。

保存キーは以下です。

```text
hatakeNoteLocal.schedules.v1
```

写真本体は `localStorage` ではなく IndexedDB に保存します。

IndexedDB のDB名とオブジェクトストア名は以下です。

```text
DB名: hatakeNoteLocalDB
オブジェクトストア名: photos
```

写真データの保存対象は以下です。

- id
- workRecordId
- plotId
- blob
- createdAt

区画データの保存対象は以下です。

- 区画ID
- 区画名
- 作物名
- 植え付け日
- 収穫予定日
- 状態
- メモ
- createdAt
- updatedAt

初回起動時はサンプル区画を保存して表示します。すでに保存済みデータがある場合、サンプル区画で上書きしません。

作業記録は必ず区画IDに紐づけて保存します。保存対象は以下です。

- id
- plotId
- date
- workType
- memo
- photoIds
- createdAt
- updatedAt

育成スケジュールは必ず区画IDに紐づけて保存します。保存対象は以下です。

- id
- plotId
- title
- date
- memo
- done
- createdAt
- updatedAt

## 注意点

- `localStorage` 保存なので、ブラウザデータを削除すると区画データも消える可能性があります。
- 端末変更や別ブラウザへの自動引き継ぎはできません。
- 写真削除、バックアップ出力・読み込みは実装済みです。クラウド同期はまだ実装していません。
- 写真機能はスマホ実機での確認が必要です。

## Ver.1.0 試作3 メモ

試作3では、区画管理と作業記録に加えて、育成スケジュール機能を追加しました。

育成スケジュールは区画に紐づき、`localStorage` の `hatakeNoteLocal.schedules.v1` に保存します。

完了チェックに対応し、チェックを変更すると `done` が更新されます。

ホーム画面には未完了の近日予定を表示します。

区画詳細画面には、その区画に紐づく未完了予定を表示します。

試作3時点では、写真記録はまだ本格実装していません。画面上には仮表示の枠だけを残していました。

## Ver.1.0 試作4 メモ

試作4では、写真記録機能を追加しました。

写真は作業記録に紐づき、作業記録は区画に紐づきます。

写真本体は IndexedDB の `hatakeNoteLocalDB` / `photos` に保存します。

作業記録には `photoIds` を保存します。

1つの作業記録につき写真は最大3枚までです。

保存前に画像を最大1200px、JPEG品質0.8程度に縮小します。

試作4時点では、写真削除はまだ未実装でした。

スマホ実機では、写真選択、プレビュー、保存後の表示、容量感を確認する必要があります。

## Ver.1.0 試作5 メモ

試作5では、編集・削除機能を追加しました。

作業記録の日付、区画、作業内容、メモを編集できます。既存写真は維持されます。

作業記録の削除に対応しました。作業記録を削除すると、紐づく `photoIds` を確認し、IndexedDB の `photos` ストアから該当写真も削除します。

区画詳細画面の写真記録枠で、写真単体削除に対応しました。写真単体を削除すると、IndexedDB の `photos` から該当写真を削除し、作業記録側の `photoIds` からも該当IDを削除します。

育成スケジュールの区画、作業予定名、予定日、メモ、完了状態を編集できます。

育成スケジュールの削除に対応しました。

区画削除はまだ未実装です。

## GitHub Pages 更新時のキャッシュ対策

GitHub Pages更新時のキャッシュ対策として、`style.css`、`data.js`、`app.js` の読み込みURLにバージョン用クエリを付与しています。

Ver.1.1 試作1では、`style.css?v=11p1`、`data.js?v=11p1`、`app.js?v=11p1` を使用しています。

## Ver.1.1 試作1 メモ

Ver.1.1 試作1では、ホーム画面に畑レイアウトを追加しました。

畑レイアウトは4×4グリッドです。

各マスに登録済み区画または空きを割り当てられます。

割り当て済みのマスをタップすると、その区画詳細画面へ移動します。

ホーム画面の「レイアウト編集」から、各マスの割り当てを変更できます。

レイアウトは `localStorage` の `hatakeNoteLocal.layout.v1` に保存します。

既存の区画・作業記録・予定・写真の保存キーは変更していません。

4×4以外のカスタム、マス結合、ドラッグ操作は未実装です。

## Ver.1.1 試作2 メモ

Ver.1.1 試作2では、栽培計画タイムライン機能を追加しました。

栽培計画は `localStorage` の `hatakeNoteLocal.cropPlans.v1` に保存します。

栽培計画は、追肥・草取り・収穫確認などを管理する育成スケジュールとは別機能です。

栽培計画画面では、区画ごとの作付け予定を月別タイムラインで確認できます。

月別タイムラインは、今月から12か月分を横スクロールで表示します。

ホーム画面に、今日以降に開始する今後の栽培計画を表示します。

GitHub Pagesのキャッシュ対策クエリを `v=11p2` に更新しました。

既存保存キーは変更していません。

## Ver.1.2 試作1 メモ

Ver.1.2 試作1では、バックアップ出力・読み込み機能を追加しました。

## Ver.1.3 試作1 メモ

Ver.1.3 試作1では、畑レイアウトの自由編集機能を追加しました。

複数マスをまとめて1つの作付けエリアとして表示できます。

`hatakeNoteLocal.layout.v2` がない場合は、初回表示時に `hatakeNoteLocal.layout.v1` から移行します。

バックアップ対象に `layout.v2` を追加しました。

GitHub Pagesのキャッシュ対策クエリを `v=13p1` に更新しました。

既存保存キーは変更していません。

ドラッグ操作や4×4以外の行列変更は未実装です。

## Ver.1.4 試作1 メモ

Ver.1.4 試作1では、作付けサイクル管理機能を追加しました。

- 既存の栽培計画 `hatakeNoteLocal.cropPlans.v1` を活かし、保存キーは変更していません。
- 栽培計画に任意項目として `plantingMethod` と `prepDaysBeforeStart` を追加しました。
- 撤去期限は保存せず、同じ区画の次の栽培計画から自動計算します。
- 計算式は「現在の作物の撤去期限 = 次の作物の開始日 - 次の作物の準備日数」です。
- ホーム画面に「撤去期限が近い区画」セクションを追加しました。
- 区画詳細画面に「作付けサイクル」セクションを追加しました。
- 栽培計画タイムラインに準備期間と撤去期限の表示を追加しました。
- バックアップ機能は既存の `cropPlans` をそのまま含めるため、追加項目も自然にバックアップ対象になります。
- GitHub Pagesキャッシュ対策クエリを `v=14p1` に更新しました。
- 既存の区画管理、作業記録、写真記録、育成スケジュール、畑レイアウト、栽培計画、バックアップ機能の保存キーは変更していません。
- ドラッグ操作、4×4以外の畑レイアウト変更、クラウド同期、ログイン、夫婦共有は未実装です。
