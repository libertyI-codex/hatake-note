# 畑ノート ローカル版 Ver.1.0正式版候補

## アプリ名

畑ノート ローカル版 Ver.1.0正式版候補

## 目的

家庭菜園・畑・花壇を、スマホまたはPCブラウザで区画ごとに管理するためのローカルWebアプリです。

データはクラウドではなく、使用しているブラウザ内に保存します。

## 実装済み機能

- ホーム画面
- 畑区画一覧
- 区画詳細
- 区画追加
- 区画編集
- 作業記録追加
- 作業記録のホーム表示
- 作業記録の区画詳細表示
- 育成スケジュール追加
- 育成スケジュール一覧
- 完了チェック
- ホーム画面への近日予定表示
- 区画詳細画面へのその区画の予定表示
- 写真選択
- 写真プレビュー
- 写真縮小保存
- 作業記録への写真紐づけ
- 区画詳細の写真記録表示

## 保存仕様

- 区画データは `localStorage` に保存します。
- 作業記録は `localStorage` に保存します。
- 育成スケジュールは `localStorage` に保存します。
- 写真本体は IndexedDB に保存します。
- 作業記録には `photoIds` を保存します。
- 写真は1記録につき最大3枚までです。
- 写真は保存前に縮小されます。

localStorage の保存キー:

- `hatakeNoteLocal.plots.v1`
- `hatakeNoteLocal.workRecords.v1`
- `hatakeNoteLocal.schedules.v1`

IndexedDB:

- DB名: `hatakeNoteLocalDB`
- オブジェクトストア名: `photos`

## 起動方法

`index.html` をブラウザで開けば使えます。

PCでは、エクスプローラーから `index.html` をダブルクリックして開けます。

## 注意点

- ローカル保存のため、同じ端末・同じブラウザ内でのみデータが保持されます。
- ブラウザデータを削除すると記録が消える可能性があります。
- スマホ機種変更時には自動移行されません。
- 妻のスマホとの共有はできません。
- クラウド保存は未実装です。
- スマホ実機確認は未実施です。
- 写真削除機能は未実装です。
- バックアップ出力機能は未実装です。

## GitHub Pagesで公開する場合の注意点

- GitHub Pagesで公開すると、スマホのブラウザからURLで畑ノートを開けます。
- ただし、データは端末ごとのブラウザ内に保存されます。
- PCで入力したデータはスマホへ自動同期されません。
- スマホで入力したデータはスマホ側のブラウザに保存されます。
- ブラウザデータを削除すると記録が消える可能性があります。
- 共有や同期を行うには、将来的にバックアップ機能またはFirebase版が必要です。
